Step 1: Split AppContext — Hook Extraction

Breaking the 999-line AppContext.tsx into focused, testable hooks

Why this first

AppContext.tsx contains ~90 state variables and ~30 handler functions in a single file. Every component that consumes one state value re-renders when any other state changes. Extracting focused hooks makes subsequent work (error handling, Polish) safer and easier.

Key rules

Do

  • Each hook manages ONE domain's state and handlers
  • Hooks call apiGet/apiPost from utils/api.ts
  • Hooks accept shared deps (currentUser, currentCompany) as params or use other hooks
  • Same localStorage keys (hub_user, hub_company, etc.) — zero behavior change
  • useEffect data-fetching chains preserved exactly

Don't

  • Don't change any API calls, types, or localStorage keys
  • Don't change UI components — they still use useAppContext()
  • Don't introduce new dependencies
  • Don't optimise prematurely (no useMemo/useCallback unless needed)
