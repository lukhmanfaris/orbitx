Recommended sequence

1
Split AppContext first — before adding anything. A 999-line monolithic context makes every subsequent change fragile. Extract into use-media, use-articles, use-workspace, use-onboarding hooks with clear boundaries.
2
Error handling + ProjectType together — error handling is cross-cutting (toasts, error boundaries, loading states), so it benefits from the clean hook structure from step 1. The ProjectType selector is a small, self-contained feature that can be done in parallel.
3
UI/UX polish last — cosmetic improvements are safest to apply once the code structure and behavior are solid. Includes loading skeletons, empty states, responsive fixes, accessibility.
